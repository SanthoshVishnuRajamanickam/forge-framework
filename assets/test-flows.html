10 Test Tiers

Static pre-unit

Lint, type-check, MISRA, SAST. Earliest gate — catches issues before tests even run.
CLI
eslint, pylint, clippy, mypy, MISRA checker, Polyspace
Runs: APPLY (before unit, per-task) | Gate: task completion

Unit per-task

Single function or module. Fast feedback. Runs after every task qualifies in APPLY.
CLI
npm test, pytest, go test, ctest
Coverage
Configurable threshold (e.g. 80%). Blocks below.
Runs: APPLY (after each task) | Gate: task completion

Integration per-checkpoint

Cross-module, service boundaries, real database. Runs at plan checkpoints.
CLI
pytest tests/integration/, Pact contracts
MCP
PSIX MCP for Qt service integration
Hooks
pre_test: start DB, seed data. post_test: teardown.
Runs: APPLY (at checkpoints) | Gate: checkpoint clearance

E2E per-phase

Full user journey through the app. Browser, mobile, or desktop.
Skill
/playwright-cli for browser automation
MCP
Playwright MCP (AI drives browser directly)
CLI
Espresso, Qt Test, Detox scripts
Target
host | simulator | device | cloud-farm
Runs: VERIFY phase | Gate: phase completion

MCP-Driven AI-as-tester

AI reads acceptance criteria, uses MCP tools to test. No scripts needed. The AC are the test spec.
Strategies
ac-driven: Given/When/Then → MCP calls
scripted: fixed tool sequence
exploratory: AI explores freely, reports anomalies
Evidence
Screenshots, DOM snapshots, network logs
Runs: VERIFY or on-demand | Gate: phase completion

Visual per-phase

Screenshot diff, snapshot testing, golden image comparison across devices/OS versions.
CLI
Jest snapshots, Percy CLI, Chromatic
Skill
Custom visual diff skills
Evidence
Before/after screenshots, diff images
Runs: VERIFY phase | Gate: phase completion

Performance per-phase

Load testing, latency budgets, Lighthouse scores, embedded timing assertions.
CLI
k6, Artillery, Lighthouse CI
Timing
gpio_toggle <100µs, API p95 <200ms
Thresholds
Configurable per-metric pass/fail gates
Runs: VERIFY or checkpoint | Gate: phase completion

Security per-phase

SAST, DAST, dependency scanning, secrets detection. Blocks on critical vulnerabilities.
CLI
npm audit, Snyk, OWASP ZAP, gitleaks
MCP
Custom security scanning MCP servers
Runs: VERIFY phase | Gate: phase completion

Platform per-milestone

Cross-system testing. Bind any number of platform executors — browsers, Qt, mobile, HIL hardware, custom MCP servers.
MCP: PSIX
Qt desktop apps, Bluetooth, device sync
MCP: dSPACE
HIL: inject signals, verify timing
Cloud farm
BrowserStack, Firebase Test Lab, AWS Device Farm
CLI
Custom harness, ./gradlew connectedTest
Runs: milestone or on-demand | Gate: milestone

Manual human-guided

Exploratory testing, accessibility audits, device matrix. FORGE generates checklists from AC.
Checklist
Auto-generated from acceptance criteria
Categories
Exploratory, accessibility, edge cases
Device matrix
Repeat per device (desktop, phone, tablet)
Runs: VERIFY phase | Gate: phase completion

Example Test Profiles

Web SaaS (Node.js + React)

unit integration mcp-driven manual 
unit:
  - type: cli
    command: "npx vitest run"
    coverage_min: 80
integration:
  - type: cli
    command: "npx vitest run tests/integration/"
e2e:
  - type: mcp
    server: playwright
    strategy: ac-driven
manual:
  - generate_checklist: true
    categories: [accessibility]

Qt + Android Cross-Platform

unit mcp platform 
unit:
  - type: cli, command: "ctest"
  - type: cli, command: "./gradlew test"
integration:
  - type: mcp
    server: psix
    tools: [psix_send_signal, ...]
platform:
  - name: "Qt via PSIX"
    type: mcp, server: psix
  - name: "Android via Appium"
    type: cli, command: "./gradlew connectedTest"
  - name: "Bluetooth sync"
    type: mcp, server: psix
  - name: "HIL via dSPACE"
    type: mcp, server: dspace

Python API Backend

unit integration e2e 
unit:
  - type: cli
    command: "pytest tests/unit/"
    coverage_min: 90
integration:
  - type: cli
    command: "pytest tests/integration/"
e2e:
  - type: skill
    skill: "/playwright-cli"
    args_template: "test {{test_scope}}"

Embedded + HIL

unit platform manual 
unit:
  - type: cli
    command: "ctest --test-dir build"
platform:
  - name: "dSPACE HIL"
    type: mcp
    server: dspace
    tools: [dspace_load_model,
      dspace_inject_signal,
      dspace_read_output,
      dspace_verify_timing]
    strategy: ac-driven
manual:
  - generate_checklist: true
    device_matrix: [target-hw-v2]
